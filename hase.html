EH:historia HASE

HASE

Historical Activity and Source Exchange — ein offenes Austauschformat für historische Handlungen und Quellen

Was ist HASE?

HASE (Historical Activity and Source Exchange) ist ein Austauschformat für historische Handlungen und Quellen. Für den Datentransfer mit EH:historia ist HASE in zwei Formaten implementiert: TEI/XML und JSON. So können Digitale Editionen und andere digitale Ressourcen ihre Daten unkompliziert in EH:historia einbringen.

HASE modelliert historische Handlungen (Activities) und die Quellen (Sources), die sie bezeugen: Briefkorrespondenz, Begegnungen, Reisen, Übernachtungen, Lektüre und mehr — aus Briefeditionen, Tagebüchern und anderen edierten Quellen.

Designprinzipien

EinfachEine JSON-Datei pro Projekt — wenige Pflichtfelder
ZweigeteiltActivities (was ist passiert?) + Sources (woher wissen wir das?)
NormdatenGND-IDs für Personen und Orte — automatische Verknüpfung
ZweisprachigLabels und Beschreibungen in Englisch und Deutsch
ErweiterbarEigene Handlungs- und Quellentypen vorschlagen (status: "proposed")
CIDOC-CRMKonform mit dem internationalen Standard für Kulturgut-Dokumentation

Dateistruktur

Eine HASE-Datei besteht aus bis zu fünf Abschnitten:

project Pflicht

Metadaten zum liefernden Projekt.

FeldPflichtBeschreibung
id✓Eindeutiger Projektschlüssel (a-z, 0-9, Bindestriche, 3–32 Zeichen)
title✓Anzeigename
descriptionKurze Beschreibung
urlURL der Projektwebseite
licenseLizenz (empfohlen: CC-BY-4.0)
contactKontakt-E-Mail
editionsUntergliederung in Editionen (Array von {id, title, url})
activityTypes Optional

Verzeichnis der verwendeten Handlungstypen. Standard-Typen können zur Vollständigkeit aufgeführt werden; neue Typen müssen hier mit status: "proposed" deklariert werden. 

"activityTypes": {
  "LETTER_SENDING": {
    "label": { "en": "Letter sending", "de": "Briefversand" },
    "context": [{ "type": "CORRESPONDENCE",
                  "label": { "en": "Correspondence", "de": "Korrespondenz" } }]
  },
  "THEATER_VISIT": {
    "label": { "en": "Theater visit", "de": "Theaterbesuch" },
    "status": "proposed",
    "description": { "en": "Attendance at a theater performance.",
                     "de": "Besuch einer Theateraufführung." }
  }
}

Das context-Feld definiert, zu welchen Zusammenhangstypen eine Handlung gehören kann. Labels sind zweisprachig ({en, de}).

sourceTypes Optional

Verzeichnis der verwendeten Quellentypen.

"sourceTypes": {
  "LETTER":      { "label": { "en": "Letter", "de": "Brief" } },
  "DIARY_ENTRY": { "label": { "en": "Diary entry", "de": "Tagebucheintrag" } }
}
activities Pflicht

Die historischen Handlungen des Projekts.

Pflichtfelder

FeldTypBeschreibung
idstringEindeutige ID innerhalb des Projekts
typestringTyp-ID (z.B. LETTER_SENDING, ENCOUNTER_PARTICIPATION)

Optionale Felder

FeldTypBeschreibung
datestring | objectDatum (siehe unten)
actorobjectBeteiligte Person oder Gruppe ({type, label, gnd})
placeobjectOrt der Handlung ({label, gnd})
thingobjectHistorisches Objekt (sourceRef oder inline)
contextobjectZusammenhang ({id, type}) — z.B. Korrespondenz, Reise
attestedBySourcearrayIDs der bezeugenden Quellen
continuedarrayIDs von Handlungen, die diese fortsetzt (P134)
wasContinuedByarrayIDs von Handlungen, die diese fortsetzen (P134i)

Datumsformate

// Exaktes Datum (Kurzform)
"date": "1774-08-16"

// Zeitpunkt mit Qualifikator (frühestens/spätestens)
"date": { "value": "1774-08-20", "qualifier": "latest" }

// Zeitspanne
"date": { "from": "1786-09-11", "to": "1786-09-12" }

qualifier: exact (Standard), earliest (frühestens), latest (spätestens).

sources Optional

Überlieferte Quellen (Briefe, Tagebucheinträge etc.), die Handlungen bezeugen.

FeldPflichtBeschreibung
id✓Eindeutige ID der Quelle
type✓Quellentyp (z.B. LETTER, DIARY_ENTRY)
archiveAufbewahrende Institution ({label, gnd})
shelfmarkSignatur im Archiv
iiifURL zum IIIF-Manifest
referencesInhaltliche Bezüge (Personen, Orte, Werke, Sachbegriffe)
attestsActivityIDs der bezeugten Handlungen
editionEditions-ID (wenn project.editions definiert)
editionUrlPermalink zur Quelle in der digitalen Edition
catalogUrlURL des Katalogeintrags (z.B. Kalliope)

Verarbeitung

EditionHub verarbeitet HASE-Dateien automatisch: 

HASE-Datei (JSON)
    |
    +-- Validierung (JSON-Schema)
    |
    +-- Normdaten-Aufloesung
    |       GND --> EditionHub-Index (automatischer Match)
    |       Neue Personen/Orte --> Vorschlag zur Aufnahme
    |       Koordinaten aus GeoNames ergaenzen
    |
    +-- RDF-Generierung (CIDOC-CRM)
    |       Activity --> crm:E7_Activity
    |       Actor    --> crm:P14_carried_out_by + owl:sameAs
    |       Place    --> crm:P7_took_place_at
    |       Thing    --> crm:P16_used_specific_object
    |       Source   --> crm:E31_Document + crm:P70_documents
    |
    +-- Upload nach RDF4J
            --> Live auf EditionHub
