HAIF

Historical Activity Interchange Format — ein offenes Austauschformat für historische Handlungen

Was ist HAIF?

HAIF (Historical Activity Interchange Format) ist ein einfaches JSON-Format, mit dem Editionsprojekte ihre Forschungsdaten an EditionHub liefern können — ohne XSLT, ohne OpenRefine, ohne RDF-Kenntnisse.

HAIF modelliert beliebige historische Handlungen: Briefsendungen, Begegnungen, Reisen, Lektüren, Aufführungen und mehr. Personen und Orte werden über Normdaten (GND, VIAF, Wikidata) referenziert — das ermöglicht automatische projektübergreifende Verknüpfungen.

Designprinzipien

Einfach~5 Felder pro Handlung — eine JSON-Datei pro Projekt
NormdatenGND/VIAF/Wikidata für Personen und Orte — automatische Verknüpfung
Kein VolltextNur strukturierte Handlungsdaten, kein TEI-Text nötig
ErweiterbarKernkatalog von Handlungstypen + eigene Typen vorschlagen
DezentralProjekte hosten ihre Datei, EditionHub harvested regelmäßig

Minimalbeispiel

Die einfachste gültige HAIF-Datei — ein Projekt mit einer einzigen Briefsendung in 18 Zeilen: 

{
  "haif": "0.1",
  "project": {
    "id": "mein-projekt",
    "title": "Mein Editionsprojekt",
    "license": "CC-BY-4.0"
  },
  "activities": [
    {
      "id": "brief-001",
      "type": "BRIEFSENDUNG",
      "date": "1786-09-03",
      "persons": [
        { "label": "Goethe, Johann Wolfgang von", "gnd": "118540238" }
      ],
      "places": [
        { "label": "Karlsbad", "gnd": "4029702-0" }
      ]
    }
  ]
}

Dateistruktur

Eine HAIF-Datei besteht aus drei Abschnitten:

project Pflicht

Metadaten zum Editionsprojekt.

FeldPflichtBeschreibung
id✓Eindeutiger Projektschlüssel (ASCII, Kleinbuchstaben, Bindestriche)
title✓Anzeigename
license✓Lizenz (empfohlen: CC-BY-4.0 oder CC0)
urlURL des Editionsprojekts
contactKontakt-E-Mail
types Optional

Eigene Aktivitätstypen, die nicht im Kernkatalog enthalten sind. Vorgeschlagene Typen werden nach Prüfung in den Katalog aufgenommen. 

"types": [
  {
    "id": "VORLESUNG",
    "label": "Vorlesung",
    "broader": "ACTIVITY",
    "description": "Eine Person liest einem Publikum vor"
  }
]
activities Pflicht

Die Handlungen des Projekts.

Pflichtfelder

FeldTypBeschreibung
idstringEindeutige ID innerhalb des Projekts
typestringTyp-ID aus Kernkatalog oder types-Array
datestringYYYY-MM-DD, YYYY-MM oder YYYY

Optionale Felder

FeldTypBeschreibung
dateApproxbooleantrue = Terminus post quem (Datum ist ein "ab"-Datum)
personsarrayBeteiligte Personen (s.u.)
placesarrayOrte der Handlung (s.u.)
objectobjectVerknüpftes materielles Objekt (Brief, Manuskript, ...)
continuesarrayIDs von Handlungen, die diese fortsetzt
continuedByarrayIDs von Handlungen, die diese fortsetzen
sourceUrlstringURL zur Handlung in der Quelledition

Person

{ "label": "Goethe, Johann Wolfgang von", "gnd": "118540238" }

label ist Pflicht. Mindestens eine Normdaten-ID (gnd, viaf, wikidata) ist dringend empfohlen — ohne ID keine automatische Verknüpfung.

Ort

{ "label": "Weimar", "gnd": "4065105-8" }

gnd, geonames oder wikidata optional. Koordinaten (lat/lng) werden automatisch ergänzt.

Objekt (optional)

{
  "type": "BRIEF",
  "archive": { "label": "GSA Weimar", "gnd": "5080444-3" },
  "shelfmark": "GSA 28/5, Bl. 1",
  "catalog": "https://...",
  "iiif": "https://...",
  "features": [
    { "label": "Werther", "type": "werk", "gnd": "4099312-4" }
  ]
}

Kernkatalog der Aktivitätstypen

Diese Typen sind vordefiniert und sofort nutzbar. Projekte können über das types-Array eigene Typen vorschlagen.

Einzelhandlungen

IDLabelZusammenhangBeschreibung
BRIEFSENDUNGBriefsendungKorrespondenzPerson sendet einen Brief
BRIEFEMPFANGBriefempfangKorrespondenzPerson empfängt einen Brief
BEGEGNUNGSTEILNAHMEBegegnungsteilnahmeBegegnungPerson nimmt an Begegnung teil
REISESTATIONReisestationReiseStation einer Reise
UEBERNACHTUNGÜbernachtung—Person übernachtet an einem Ort
LEKTUERELektüre—Person liest ein Werk

Zusammenhänge (Ereignisse)

IDLabelKomponentenBeschreibung
KORRESPONDENZKorrespondenzBriefsendung + BriefempfangBriefwechsel als Einheit
BEGEGNUNGBegegnungBegegnungsteilnahmeTreffen von Personen
REISEReiseReisestationReise von A nach B

Eigene Typen vorschlagen: Definieren Sie neue Typen im types-Array Ihrer HAIF-Datei. Nach Prüfung durch das EditionHub-Team werden sie in den Kernkatalog aufgenommen und stehen allen Projekten zur Verfügung.

Daten einreichen

1

HAIF-Datei erstellen

Kopieren Sie das Minimalbeispiel und passen Sie es an. Nutzen Sie GND-IDs für Personen und Orte, wo möglich.

2

Datei an einer stabilen URL hosten

Z.B. auf dem eigenen Server, GitHub, oder Zenodo. Die Datei bleibt bei Ihnen — EditionHub holt sie ab.

3

URL registrieren

EditionHub harvested Ihre HAIF-Datei regelmäßig. Änderungen werden automatisch übernommen.

Verarbeitung

EditionHub verarbeitet HAIF-Dateien automatisch: 

HAIF-Datei (JSON)
    |
    +-- Validierung (JSON-Schema)
    |
    +-- Normdaten-Auflösung
    |       GND/VIAF/Wikidata --> EditionHub-Index (automatischer Match)
    |       Neue Personen/Orte --> Vorschlag zur Aufnahme
    |       Koordinaten aus GeoNames ergaenzen
    |
    +-- RDF-Generierung (CIDOC-CRM)
    |       Activity --> crm:E7_Activity
    |       Person   --> crm:P14_carried_out_by + owl:sameAs
    |       Place    --> crm:P7_took_place_at
    |       Object   --> crm:P16_used_specific_object
    |
    +-- SHACL-Validierung
    |
    +-- Upload nach RDF4J
            --> Live auf EditionHub

  • RDF verstehen
